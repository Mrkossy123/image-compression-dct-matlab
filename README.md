# Image Compression with DCT and Quantization

Αυτό το repository περιλαμβάνει δύο scripts για επεξεργασία εικόνας και βασικές τεχνικές συμπίεσης με χρήση του **Discrete Cosine Transform (DCT)**.  
Η εργασία βασίζεται στην grayscale εικόνα `cameraman.tif` και εξετάζει δύο διαφορετικές προσεγγίσεις:

1. **Thresholding σε DCT coefficients** πάνω σε ολόκληρη την εικόνα.
2. **Block-based DCT quantization** σε blocks `8x8`, με πίνακα κβαντοποίησης τύπου JPEG.

Ο κώδικας έχει διατηρηθεί στην αρχική του μορφή, όπως υλοποιήθηκε για ακαδημαϊκή άσκηση, χωρίς refactoring ή αλλαγές στη δομή του.

---

## Περιεχόμενα repository

- `code1.m`  
  Εφαρμόζει 2D DCT σε ολόκληρη την εικόνα, μηδενίζει συντελεστές κάτω από συγκεκριμένα thresholds και υπολογίζει την ποιότητα αποκατάστασης.

- `code2.m`  
  Εφαρμόζει block-based DCT σε blocks `8x8`, quantization με τρεις διαφορετικές κλίμακες πίνακα κβαντοποίησης και μετρά την επίδραση στην ποιότητα της εικόνας.

- `cameraman.tif`  
  Η grayscale εικόνα εισόδου που χρησιμοποιείται και από τα δύο scripts.

## Στόχος

## Αναλυτική περιγραφή του `code1.m`

### Αντικείμενο

Το `code1.m` μελετά την επίδραση του **thresholding** στους DCT συντελεστές της εικόνας.

Η βασική ιδέα είναι η εξής:

1. Η αρχική grayscale εικόνα μετασχηματίζεται με **2D DCT**.
2. Συντελεστές με μικρό απόλυτο μέγεθος θεωρούνται λιγότερο σημαντικοί.
3. Οι συντελεστές αυτοί μηδενίζονται.
4. Η εικόνα ανακατασκευάζεται με **inverse DCT**.
5. Υπολογίζεται η απώλεια ποιότητας σε σχέση με την αρχική εικόνα.

### Thresholds που εξετάζονται

Το script εξετάζει τρεις περιπτώσεις:

- `abs(F(i,j)) < 5`
- `abs(F(i,j)) < 10`
- `abs(F(i,j)) < 15`

Για κάθε περίπτωση:

- μηδενίζονται οι αντίστοιχοι DCT coefficients,
- δημιουργείται νέα ανακατασκευασμένη εικόνα,
- υπολογίζεται **MSE**,
- υπολογίζεται **PSNR**,
- υπολογίζεται ο αριθμός των μηδενικών στοιχείων στον μετασχηματισμένο πίνακα.

### Τι εμφανίζει

Το script εμφανίζει:

- την αρχική εικόνα,
- την ανακατασκευασμένη εικόνα για threshold `< 5`,
- την ανακατασκευασμένη εικόνα για threshold `< 10`,
- την ανακατασκευασμένη εικόνα για threshold `< 15`.

### Τι τυπώνει στο command window

Για κάθε threshold εμφανίζονται:

- `MSE`
- `PSNR`
- αριθμός μηδενικών DCT coefficients

### Ερμηνεία αποτελεσμάτων

Όσο αυξάνεται το threshold:

- περισσότεροι DCT coefficients γίνονται μηδέν,
- άρα μειώνεται η πληροφορία που διατηρείται,
- και η εικόνα τείνει να χάνει λεπτομέρεια,
- κάτι που συνήθως αυξάνει το **MSE** και μειώνει το **PSNR**.

Με άλλα λόγια, το script δείχνει το κλασικό trade-off ανάμεσα σε **συμπίεση/απλοποίηση** και **ποιότητα ανακατασκευής**.

## Αναλυτική περιγραφή του `code2.m`

### Αντικείμενο

Το `code2.m` υλοποιεί μια **block-based** προσέγγιση DCT, εμπνευσμένη από τη λογική της JPEG συμπίεσης.

Η εικόνα χωρίζεται σε blocks `8x8`, και σε κάθε block εφαρμόζεται DCT. Στη συνέχεια οι DCT coefficients κβαντίζονται με έναν πίνακα κβαντοποίησης και μετά ανακατασκευάζονται με inverse βήματα.

### Βασικά βήματα

Για κάθε block `8x8`:

1. εφαρμόζεται `dct2`,
2. γίνεται διαίρεση με τον πίνακα κβαντοποίησης,
3. γίνεται `round` για να προκύψουν οι quantized coefficients,
4. υπολογίζεται entropy στους απόλυτους quantized coefficients,
5. γίνεται inverse quantization μέσω πολλαπλασιασμού με τον ίδιο πίνακα,
6. εφαρμόζεται `idct2`,
7. συντίθεται η τελική ανακατασκευασμένη εικόνα.

### Πίνακες κβαντοποίησης που εξετάζονται

Το script ξεκινά από έναν αρχικό πίνακα `Q1` και εξετάζει τρεις περιπτώσεις:

- `Q1 = Q`
- `Q2 = 2 * Q1`
- `Q3 = 4 * Q1`

Η αύξηση του πίνακα κβαντοποίησης σημαίνει πιο επιθετική κβαντοποίηση και συνήθως μεγαλύτερη απώλεια πληροφορίας.

### Τι μετρά

Το script υπολογίζει:

- **entropy** της αρχικής εικόνας,
- **entropy** των `abs(F_blocks)` μετά την κβαντοποίηση,
- **MSE** μεταξύ αρχικής και ανακατασκευασμένης εικόνας,
- **PSNR** για κάθε περίπτωση quantization.

### Τι εμφανίζει

Το script εμφανίζει:

- την αρχική εικόνα,
- την ανακατασκευασμένη εικόνα για `Q1`,
- την ανακατασκευασμένη εικόνα για `Q2 = 2*Q1`,
- την ανακατασκευασμένη εικόνα για `Q3 = 4*Q1`.

### Ερμηνεία αποτελεσμάτων

Καθώς ο πίνακας quantization γίνεται μεγαλύτερος:

- περισσότεροι DCT coefficients στρογγυλοποιούνται σε μικρότερες ή μηδενικές τιμές,
- μειώνεται η λεπτομέρεια της εικόνας,
- η ανακατασκευή γίνεται πιο απωλεστική,
- το **MSE** τείνει να αυξάνεται,
- και το **PSNR** τείνει να μειώνεται.

Αυτό προσομοιώνει τη βασική αρχή της απωλεστικής συμπίεσης με DCT και quantization.

## Μετρικές που χρησιμοποιούνται

### MSE (Mean Squared Error)

Το **MSE** μετρά το μέσο τετραγωνικό σφάλμα μεταξύ της αρχικής και της ανακατασκευασμένης εικόνας.

Μικρότερο MSE σημαίνει ότι η ανακατασκευασμένη εικόνα είναι πιο κοντά στην αρχική.

### PSNR (Peak Signal-to-Noise Ratio)

Το **PSNR** είναι μια κλασική μετρική αξιολόγησης στην επεξεργασία εικόνας και βασίζεται στο MSE.

Μεγαλύτερο PSNR σημαίνει καλύτερη ποιότητα ανακατασκευής.

### Entropy

Στο `code2.m` χρησιμοποιείται entropy:

- μία φορά στην αρχική εικόνα,
- και στη συνέχεια στους απόλυτους quantized coefficients.

Η entropy εδώ χρησιμοποιείται ως ένδειξη της κατανομής / πολυπλοκότητας των δεδομένων μετά την κβαντοποίηση.

## Περιορισμοί

Το repository έχει καθαρά εκπαιδευτικό σκοπό και παρουσιάζει μια βασική υλοποίηση των ιδεών DCT thresholding και block quantization. Δεν περιλαμβάνει:

- πλήρη υλοποίηση JPEG bitstream,
- zig-zag scan,
- run-length encoding,
- entropy coding / Huffman coding,
- γενικευμένη διαχείριση οποιασδήποτε εικόνας εισόδου,
- refactoring σε modular functions.

Άρα, το repository πρέπει να αντιμετωπίζεται ως **εκπαιδευτική άσκηση / demonstrator** και όχι ως production-ready σύστημα συμπίεσης εικόνας.
